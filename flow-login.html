🔐 Login & Firebase Auth

Datenfluss von der Anmeldung bis zur fertigen App-Session

Frontend (Browser)
Firebase Backend
1 Nutzer gibt E-Mail + Passwort ein und klickt „Anmelden"
auth.signInWithEmailAndPassword(email, pw)
▶
🔑
Firebase Auth prüft E-Mail + Passwort gegen die Auth-Datenbank
2
auth.onAuthStateChanged(fbUser) wird automatisch ausgelöst
◀
fbUser { uid, email, displayName }
✓
Auth erfolgreich → gibt fbUser Objekt mit uid zurück
3 Profildaten aus Datenbank laden (uid bekannt)
db.collection("users").doc(uid).get()
▶
🗄️
Firestore: users/
Sucht Dokument mit dieser UID
4
S.user = { name, u, email, avatar, id } gesetzt
◀
DocumentSnapshot: { name, u, avatar, email }
📄
Gibt vollständiges Nutzerprofil zurück
⚡ Promise.all() — 3 Anfragen gleichzeitig
5 App-Daten parallel laden: Buchungen, Freundschaftsanfragen, Gruppen
bookings.where("uid").get()
friendRequests.where("from"/"to").get()
groups.where("members").get()
▶
🗄️
Firestore:
bookings/ · friendRequests/ · groups/
3 Collections gleichzeitig
6
S.bookings[], _friendReqs[], _groups[] befüllt
◀
3× QuerySnapshot mit allen Dokumenten
✓
Alle Snapshots zurückgegeben
🎉
initApp() — Startseite wird angezeigt, Nutzer ist eingeloggt
Fehlerfall: Falls E-Mail oder Passwort falsch → Firebase Auth wirft einen Fehler-Code z.B. auth/wrong-password oder auth/user-not-found. Das Frontend fängt diesen ab und zeigt eine deutsche Fehlermeldung. Kein Firestore-Zugriff findet statt.
