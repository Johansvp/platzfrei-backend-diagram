📡 Wie werden die Daten übertragen?
🔒
Verschlüsselung (TLS/HTTPS)
Alle Anfragen laufen über HTTPS. Die Verbindung zwischen App und Firebase ist mit TLS verschlüsselt. Keine Daten im Klartext.
📦
Format (JSON)
Daten werden als JSON übertragen. Das Firebase SDK wandelt JS-Objekte automatisch um. Bei Lesen kommen die Daten als DocumentSnapshot zurück.
⚡
Protokoll (gRPC / REST)
Firestore nutzt intern gRPC für schnelle bidirektionale Kommunikation. Im Browser wird automatisch auf WebSocket / HTTP/2 zurückgegriffen.
🔄 Die 4 wichtigsten Abläufe im Detail
🔐 Login & Session
1
Nutzer gibt E-Mail + Passwort ein
2
auth.signInWithEmailAndPassword() → Firebase Auth
3
auth.onAuthStateChanged() empfängt den Nutzer automatisch
4
db.collection("users").doc(uid).get() lädt das Profil
5
Buchungen, Freunde & Gruppen werden parallel geladen
📅 Platz buchen
1
Nutzer wählt Stadt, Platz, Dauer & Uhrzeit
2
db.collection("bookings").add(bkData) → Dokument-ID zurück
3
Lokal in S.bookings[] gespeichert (kein erneuter Load)
4
Eingeladene Freunde: Nachricht via chats/ oder groupChats/
5
Bestätigungsdialog wird lokal gerendert
👥 Freundschaftsanfrage
1
Nutzer sucht: db.collection("users").get() → Filtert lokal
2
Anfrage: db.collection("friendRequests").add({from, to, status:"pending"})
3
Empfänger sieht Anfrage beim nächsten Laden (alle 30 Sek. Refresh)
4
Annehmen: .doc(id).update({status:"accepted"})
5
Ablehnen / Entfernen: .doc(id).delete()
💬 Nachricht senden
1
Chat öffnen: Key = [uid_A, uid_B].sort().join("_")
2
Laden: db.collection("chats").doc(key).get()
3
Neue Nachricht wird an messages[] Array angehängt
4
Speichern: db.collection("chats").doc(key).set({messages, updatedAt})
5
Lokaler Cache in _realChats{} bleibt aktuell
